Trinity Framework

VERSION: 1.0.0 | STATUS: Production Ready | LICENSE: MIT

Trinity Framework creates context-resilient AI agent workspaces. No infrastructure required. Your agent maintains identity, remembers decisions, and picks up where you left off — every session.

Why Trinity?

Persistent Identity

Your agent knows who it is, what project it's working on, and its role. No more re-explaining context every conversation.

Context Resilience

Survive context window resets. AUTO-COMPACT.md ensures your agent recovers gracefully when limits are hit.

Decision Boundaries

Clear authority levels prevent your agent from making decisions it shouldn't. Ask first, never assume.

Status Tracking

Every session ends with a structured summary. Pick up exactly where you left off, every time.

Zero Infrastructure

No databases, no servers, no complex setup. Just markdown files and your AI interface of choice.

The Protocol Files

Trinity Framework is built on three core protocol files that work together:

AGENT.md

Identity & Mission — Who is this agent? What's its role? What decisions can it make autonomously vs. what requires human approval?

STATUS.md

Current State — What's in progress? What's blocked? What was accomplished? A living document that captures project momentum.

AUTO-COMPACT.md

Recovery Protocol — When the context window fills up, this file tells the agent how to recover and continue working without losing track.

$ chmod +x setup.sh
