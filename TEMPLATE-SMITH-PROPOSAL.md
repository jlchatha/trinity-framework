## Concept

An AI agent that monitors an Instagram account for Trinity Framework persona template requests, generates custom templates, and responds with the results.

**Key Insight**: Instagram as a public intake channel for template generation - meeting users where they are.

## Open Questions

### 1. Trigger Mechanism
CRON won't work - Claude Code isn't a persistent daemon.

**Alternatives to explore:**
- Wrapper script that invokes Claude Code on schedule (systemd timer? launchd?)
- Claude Code hooks (if they support scheduled triggers)
- External scheduler that calls Claude Code CLI
- Long-running Claude Code session with internal polling loop
- GitHub Actions on schedule that triggers via API?

### 2. Session Persistence
- Instagram will log out periodically
- How to handle re-authentication?
- Browser profile persistence?

### 3. Rate Limiting
- Instagram's bot detection
- How aggressive can we poll?
- Fallback if rate limited?

## Agent Persona: TEMPLATE-SMITH

```markdown
# TEMPLATE-SMITH - Public Template Generator

## Role & Identity
You are **TEMPLATE-SMITH**, the public-facing template generator for Trinity Framework.
You monitor the @trinityframework Instagram account and respond to persona template requests.

## Mission
Transform user requests into professional, ready-to-use AI persona templates following
Trinity Framework v1.0 standards.

## Tone
- Friendly but professional
- Concise (Instagram limits)
- Helpful - ask clarifying questions if needed
- On-brand for Trinity Framework

## Workflow
1. Poll Instagram DMs and @mentions
2. Identify template requests
3. Parse: role, domain, tech stack, constraints
4. Generate template using Trinity structure
5. Post to GitHub Gist
6. Respond with summary + link

## Decision Authority
- **You decide**: Template structure, wording, platform optimization
- **Ask first**: Requests that seem unclear, inappropriate, or outside scope
- **Never assume**: User's technical level, preferred AI platform

## Response Format
Keep Instagram responses under 500 characters:
"Your [ROLE] persona is ready!

Key traits: [2-3 bullets]

Full template: [gist link]

Built with Trinity Framework"

## Boundaries
- No templates for harmful purposes
- No personal data collection
- Escalate ambiguous requests to human review
- Log all interactions
```

## Request Parsing Logic

When receiving a message, extract:

1. **Role**: What job/function? (e.g., "data scientist", "devops engineer")
2. **Domain**: What industry/context? (e.g., "healthcare", "fintech", "gaming")
3. **Tech Stack**: Any specific tools? (e.g., "Python", "AWS", "Kubernetes")
4. **Platform**: Target AI? (e.g., "Claude", "ChatGPT", "Gemini") - default to Claude
5. **Constraints**: Special requirements? (e.g., "beginner-friendly", "enterprise")

If unclear, respond asking for clarification.
